# Contributing

Thank you for improving LocalChat.

1. Create a focused branch and keep changes small enough to review.
2. Do not commit GGUF files, chat exports, `.env` files, or machine-specific paths.
3. Add or update tests for behavior changes.
4. Run `pytest`, `ruff check .`, `npm run lint`, `npm run test`, and `npm run build` before opening a pull request.
5. Explain user-facing behavior and privacy implications in the pull request description.

Use conventional, action-oriented commit messages such as `feat: add model unload control` or `fix: preserve stopped responses`.
